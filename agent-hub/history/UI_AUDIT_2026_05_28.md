  ## PLAYER FEATURES — CODED BUT NOT ACCESSIBLE FROM UI

  All 5 items below were verified by reading the full 3222-line player_screen.dart and cross-referencing every callback passed into _ControlsOverlay against the overlay's build() method.

  ### PL-001 — TransparentPlayerSlider: No Trigger Button
  - **Widget:** `lib/widgets/player/transparent_player_layer.dart`
  - **State:** `bool _showTransparentSlider = false` (line 240)
  - **Renders when:** `_prefs.transparentModeEnabled && _showTransparentSlider` (line 2238)
  - **onClose:** sets it back to false (line 2246)
  - **Problem:** `_showTransparentSlider = true` is set NOWHERE in the entire file. The widget exists and renders correctly when the flag is true, but no button, tap, or gesture ever sets it to true.
  - **Impact:** Users who enable Transparent Mode in Player Settings cannot adjust opacity.
  - **Fix:** Add a button in QuickSettingsPanel (when transparent mode is on) that calls `setState(() => _showTransparentSlider = true)`. Since QuickSettingsPanel lives in the parent stack, this needs a callback — e.g. add `onOpenTransparentSlider` to QuickSettingsPanel or expose it via the existing onChanged pattern.

  ### PL-002 — PlaybackInfoOverlay: No Toggle Button in Controls Overlay
  - **Widget:** `lib/widgets/player/playback_info_overlay.dart`
  - **State:** `bool _showPlaybackInfo = false` (line 225)
  - **Renders when:** `_showPlaybackInfo && !_showControls` (line 2050) — only visible when controls are hidden
  - **Callback:** `onTogglePlaybackInfo` IS declared in _ControlsOverlay constructor (line 2286) and IS passed in from parent (lines 1862–1864)
  - **Problem:** `onTogglePlaybackInfo` is never called from any button in `_ControlsOverlay.build()`. The right-side 9-button strip (Audio/Sub/Fit/Speed/Night/Loop/Sleep/Mark/More) and the entire bottom bar have been audited — none reference this callback.
  - **Impact:** Codec/resolution/FPS/bitrate/buffer info overlay is inaccessible.
  - **Fix:** Add to right-side strip: `_MxSideBtn(icon: Icons.info_outline_rounded, label: 'Info', onTap: onTogglePlaybackInfo)`

  ### PL-003 — VideoEnhancePanel: No Trigger Button in Controls Overlay
  - **Widget:** `lib/widgets/player/video_enhance_panel.dart`
  - **State:** `bool _showVideoEnhance = false` (line 239)
  - **Renders when:** `_showVideoEnhance` (line 2221)
  - **Callback:** `onToggleVideoEnhance` IS declared in _ControlsOverlay constructor (line 2303) and IS passed in (line 1889)
  - **Problem:** `onToggleVideoEnhance` is never called from any button in `_ControlsOverlay.build()`.
  - **Impact:** Brightness/contrast/saturation/sharpen controls are completely inaccessible.
  - **Fix:** Add to right-side strip: `_MxSideBtn(icon: Icons.auto_fix_high_rounded, label: 'Enhance', onTap: onToggleVideoEnhance)`

  ### PL-004 — SceneBookmarksPanel: View/Open Button Missing (Add-Only)
  - **Widget:** `lib/widgets/player/scene_bookmarks_panel.dart`
  - **State:** `bool _showBookmarksPanel = false` (line 218)
  - **Renders when:** `_showBookmarksPanel` (line 2189)
  - **Callbacks:** Both `onToggleBookmarks` AND `onAddBookmark` are declared in _ControlsOverlay
  - **Problem:** The "Mark" side button calls `onAddBookmark` (adds a bookmark at current position). `onToggleBookmarks` (opens the panel to view/seek/delete bookmarks) is NEVER called from any button.
  - **Impact:** Users can add bookmarks silently but cannot view, seek to, or delete them — the management panel is inaccessible.
  - **Fix (option A):** Change "Mark" button to a long-press: `onTap: onAddBookmark, onLongPress: onToggleBookmarks`
  - **Fix (option B):** Show a second side button (e.g. list icon) when `bookmarks.isNotEmpty`

  ### PL-005 — Screenshot: No Trigger Button in Controls Overlay
  - **Handler:** `_takeScreenshot()` (player_screen.dart line 581) — captures MPV frame, saves to gallery
  - **Callback:** `onTakeScreenshot` IS declared in _ControlsOverlay constructor (line 2304) and IS passed in (line 1890)
  - **Problem:** `onTakeScreenshot` is never called from any button in `_ControlsOverlay.build()`.
  - **Impact:** Screenshot-to-gallery feature is completely inaccessible.

  ### HS-001 — CatalogState.trending: Computed But Never Shown
  - **Provider:** `lib/providers/catalog_provider.dart`
  - **Field:** `List<CatalogItem> trending` — populated by `_computeTrending(movies, shows)` (line 71)
  - **Problem:** `HomeScreen._buildContent()` reads `catalog.movies`, `catalog.shows`, `catalog.recentlyWatched` — but NEVER reads `catalog.trending`. No "Trending" section exists.
  - **Impact:** The trending algorithm runs on every sync but its output is silently discarded.
  - **Fix:** Add after the "Continue Watching" section in `_buildContent()`:
  ```dart
  if (catalog.trending.isNotEmpty)
    SliverToBoxAdapter(child: _ContentSection(
      title: 'Trending Now',
      items: catalog.trending,
    )),
  ```

### New Bugs Found

#### BUG-002 — show_detail_screen.dart: `_playMovie()` Missing `content_type` in Route Args
- **File:** `raddflix_flutter/lib/screens/show_detail_screen.dart` lines 126–140
- **Severity:** Medium — functional bug affecting SmartIntroStore logic for movies
- **Issue:** `_playMovie()` pushes to the player route without a `content_type` key:
  ```dart
  Navigator.pushNamed(context, AppRoutes.player, arguments: {
    'file_id': fileId,
    'title': widget.item.title,
    'local_path': null,
    'episodes': <Map<String, dynamic>>[],
    'episode_index': 0,
    // ← 'content_type' key is absent
  });
  ```
  `app.dart` route builder uses `args['content_type'] as String? ?? 'series'`, so movies silently default to `contentType = 'series'`.
- **Impact:** `SmartIntroStore.shouldShow()` fires for movies (it checks `contentType`). Skip Intro button and auto-skip can appear on movies incorrectly.
- **Fix:** Add `'content_type': widget.item.isMovie ? 'movie' : widget.item.contentType` to the args map. `_playEpisode()` should also pass `'content_type': widget.item.contentType`.

#### BUG-003 — login_screen.dart: Logo Shows 'J' Instead of 'R' (Stale JazzMAX Branding)
- **File:** `raddflix_flutter/lib/screens/login_screen.dart` line 197
- **Severity:** Medium — visible on every login; contradicts Rule 7 (branding)
- **Issue:** The `_Logo` widget shows `Text('J')` — stale Jazz branding. Every other screen uses 'R'.
  ```dart
  child: Text('J', style: TextStyle(color: Colors.white, fontSize: 32, ...)),
  ```
- **Fix:** Change `'J'` → `'R'` on line 197 of login_screen.dart.

#### BUG-004 — subscription_screen.dart: WhatsApp Button Shows SnackBar Only (No URL Launch)
- **File:** `raddflix_flutter/lib/screens/subscription_screen.dart` line ~388
- **Severity:** High — critical user-facing action that silently fails
- **Issue:** `_buildWhatsAppButton()` `onPressed` only shows a SnackBar with a placeholder text and never calls `launchUrl()`:
  ```dart
  onPressed: () {
    ScaffoldMessenger.of(context).showSnackBar(const SnackBar(
      content: Text('WhatsApp support: Contact via RaddFlix help'),
      ...
    ));
  },
  ```
  `url_launcher` is already a dependency in pubspec.yaml but is not imported here.
- **Impact:** Users who tap "Contact Support on WhatsApp" see a toast and nothing happens. Subscription support is unreachable.
- **Fix:** Import `url_launcher`, replace the SnackBar with:
  ```dart
  final uri = Uri.parse('https://wa.me/923XXXXXXXXX?text=RaddFlix+Support');
  if (await canLaunchUrl(uri)) launchUrl(uri, mode: LaunchMode.externalApplication);
  ```
  (Replace with the actual WhatsApp number stored in `AppConstants`.)

#### BUG-005 — tid_status_screen.dart: `_buildContactSupportButton()` Has Empty onPressed
- **File:** `raddflix_flutter/lib/screens/tid_status_screen.dart` lines 416–432
- **Severity:** High — support button on payment-failed screen does nothing
- **Issue:** The "Contact Support" ElevatedButton (shown when TID verification fails) has a no-op handler:
  ```dart
  ElevatedButton.icon(
    onPressed: () {},   // ← empty, does nothing
    icon: const Icon(Icons.support_agent_rounded),
    label: const Text('Contact Support'),
    ...
  )
  ```
- **Impact:** Users whose payment fails and who tap "Contact Support" get no response. Critical support path is broken.
- **Fix:** Same as BUG-004 — launch WhatsApp URL or navigate to `subscription_screen.dart`. Example:
  ```dart
  onPressed: () async {
    final uri = Uri.parse('https://wa.me/923XXXXXXXXX?text=TID+Support+Request');
    if (await canLaunchUrl(uri)) launchUrl(uri, mode: LaunchMode.externalApplication);
  },
  ```
