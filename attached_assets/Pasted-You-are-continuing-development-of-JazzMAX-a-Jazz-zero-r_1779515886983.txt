PROJECT: JazzMAX
A streaming app for Jazz SIM users in Pakistan. All videos hosted on JazzDrive (Jazz's cloud 
storage), which is zero-rated — meaning Jazz users watch without using mobile data.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CRITICAL RULE:
JazzDrive is used ONLY for video streaming and file downloads. 
NEVER use JazzDrive for images (posters, backdrops, thumbnails).
All images must come from TMDB API or local disk cache.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CURRENT STATE (what is already working):
- Watch Prototype: Flask app running on port 8000, entry: _watch_prototype/run.py
- Radd Hub: Admin backend running on port 5000, entry: radd-hub/radd_hub.py
- Both share the same SQLite database at radd-hub/data/
- 14 movies + 4 TV shows in the database
- JazzDrive link generation works: radd-hub/hub/jazzdrive.py
  - generate_direct_link(share_url, target_filename=filename) — for specific files in folders
  - All episodes share the same folder share_url; MUST pass target_filename to get right episode
- Stream links cached 6 hours (LINK_CACHE_SECONDS = 21600 in routes/watch.py)
- Posters cached locally: _watch_prototype/posters/title_{id}.jpg and show_{file_id}.jpg
- TMDB used for all poster and backdrop images
- Web UI: JazzMAX branded app with Swiper.js 11 + GSAP 3.12, mobile-first
  - Featured showcase carousel at top (peeking cards, not full-screen hero)
  - Horizontal Swiper content rows for Movies and TV Shows
  - Bottom navigation: Home | Search | Downloads | My
  - Slide-up bottom sheet for title details + episode list
  - Full-screen video player overlay

DATABASE SCHEMA (SQLite — titles table key columns):
id, title, year, media_type (movie/show), poster, backdrop, backdrop_share_url,
rating, plot, overview, runtime, genres (JSON string), language,
trailer_url, is_free, folder_share_url, poster_share_url

Files table: id, title_id, filename, file_id, share_url, season, episode
